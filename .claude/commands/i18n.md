# i18n — Add translations to a component

Add i18n support to the component in the current IDE selection or the file path passed as an argument: $ARGUMENTS

## Project context

- Translation library: `next-intl`
- Server components use `getTranslations` from `next-intl/server` (async)
- Client components use `useTranslations` from `next-intl`
- Translation files: `frontend/src/messages/en.json` and `frontend/src/messages/vi.json`
- Locales: `en` (English) and `vi` (Vietnamese)
- Routing config: `frontend/src/i18n/routing.ts`

## Steps

1. Read the target component file to identify all hardcoded user-facing strings (labels, placeholders, button text, alt text, aria-labels, error messages, etc.).
2. Choose a namespace name that matches the component's domain (e.g. `home`, `dashboard`, `auth.signin`). Reuse an existing namespace if the component already belongs to one.
3. Add the new translation keys under that namespace in `frontend/src/messages/en.json`.
4. Add the equivalent Vietnamese translations under the same namespace in `frontend/src/messages/vi.json`.
5. Update the component:
   - If it is a server component (no `"use client"`): add `import { getTranslations } from "next-intl/server"`, make the function `async`, and call `const t = await getTranslations("namespace")`.
   - If it is a client component (`"use client"`): add `import { useTranslations } from "next-intl"` and call `const t = useTranslations("namespace")`.
6. Replace every hardcoded string with a `t("key")` call.
7. Do not change any styling, logic, or structure — only extract strings.
